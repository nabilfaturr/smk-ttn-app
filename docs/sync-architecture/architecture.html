Setiap perubahan lokal dicatat di "buku pesanan" dulu, baru sync engine eksekusi.

Kenapa outbox?
  • Atomic: insert data + catat antrian dalam 1 transaksi
  • Resilient: app crash? sync_log masih ada → retry saat restart
  • Idempotent: UNIQUE constraint → no duplicate entries

Outbox pattern = no data loss. Kalau crash saat push, sync_log masih pending → di-retry saat restart.

📋 sync_log — Outbox Pattern (Buku Pesanan)

Tabel "antrian" yang mencatat setiap perubahan lokal. Ada di src/lib/db/schema.ts (tabel ke-22).

Kenapa sync_log penting?
  • Atomic: data + antrian dalam 1 transaction
  • Crash recovery: persist saat restart
  • Idempotent: no duplicate entries
  • Retry reliable: exponential backoff otomatis
  • Quota management: batch 20 per cycle
